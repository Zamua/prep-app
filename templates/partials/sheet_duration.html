{# Shared bottom-sheet drawer for picking a snooze / mute duration. Triggered by any button with data-sheet-open="duration-sheet" + data-sheet-action="" + data-sheet-title + data-sheet-deck. The JS in modules/sheet.js handles open + populate + close + chip submit. The same
is reused for every trigger — its action attribute gets rewritten on open. Field shape posts to prep.web.durations.parse_until: - preset = one of {1h, 4h, tonight, tomorrow, 1d, 1w, forever} (Mute also offers 'forever' since indefinite mute is meaningful) - OR (custom = int, unit = hours|days|weeks) #}

{# NO data-submit-pending here — this form has multiple submitters (one per chip). submit-pending grabs the FIRST submit button and disables it on the submit event, but in a multi-submitter form the first button is often the one the user just clicked, and a disabled control is excluded from the form data set. Result: the submitter's name/value is dropped, the server gets neither preset nor (custom + unit), and the route 400s. Sheet closes on success so double-tap protection isn't load-bearing here. #} {# Wake-now chip — visible only when the trigger sets data-sheet-show-wake=1 (i.e. the sheet was opened for an already-snoozed session). Posts preset=wake which the route interprets as "clear the snooze". CSS hides .sheet-wake-only elements unless the parent dialog carries .has-wake. #}
