Export {{ deck_name }}

Send your cards somewhere.

Three formats with different trade-offs. Use .prepdeck to back up everything — including your study history — so you can restore the deck later. Use CSV or Anki if you want to share cards with someone else or move them into another app.

{# Three format cards. Each card has the format name, a one-line description, a paragraph of detail, and an Export button that the deck-export.js module hooks. The hook prefers the Web Share API on iOS so the share sheet pops cleanly instead of Safari's full-screen "Open in..." preview hijacking the PWA. #}

  • Format · backup

    .prepdeck

    Full fidelity backup. Cards, FSRS state, review history.

    Includes every prep field plus your study progress: per-card FSRS state, every review you've made, and (for trivia decks) the queue order with per-card answered state. Import back into any prep instance via New deck → Restore .prepdeck.

  • Format · cards only

    CSV

    All card fields, no study history. Anki-friendly.

    Every prep card field (prompt, answer, choices, rubric, skeleton, language, answer_regex, explanation) lands in a single CSV. Opens cleanly in Excel / Numbers / Google Sheets. Re-importable into prep, into Anki, or anywhere that reads plain CSV. Does not carry study history.

  • Format · Anki-compatible

    Anki .apkg

    Flattened to basic Anki notes. Lossy. No history.

    Imports directly into Anki Desktop / AnkiMobile / AnkiWeb. Prep flattens each card to a Basic note (Front + Back); MCQ choices become bulleted text on the front, skeleton / rubric / explanation get appended to the back as labeled sections. The original structured fields don't survive a re-import back into prep — use .prepdeck if you want lossless round-trip.
