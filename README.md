# prep

Interview-prep flashcards on your tailnet, generated by your local Claude
CLI. Spaced repetition — wrong answers come back in 10 minutes, right
ones step out (`10m → 1d → 3d → 7d → 14d → 30d`). Self-hosted, single
binary stack, no SaaS.

## What it does

- **Decks** — one per role / company / topic you're prepping for. You
  describe the deck once in plain text (links allowed; Claude will
  web-fetch them); cards generate from that description on demand.
- **Cards** — `code`, `mcq`, `multi`, or `short`. Free-text answers are
  graded by Claude against the model answer + rubric.
- **Study** — sessions resume across devices; the codemirror editor
  supports vim / emacs / vanilla; results SRS-promote in 10-min to
  30-day intervals.
- **Improve** — tap any card → free-form prompt → Claude rewrites it.
  Tap a deck → free-form prompt → Claude proposes a patch (additions /
  modifications / deletions); you preview and apply.
- **Notifications** — web push (VAPID) when cards become due. Daily
  digest or threshold-based; quiet hours; per-user opt-in.
- **PWA** — installs to iOS / Android / desktop home screen with the
  Fraunces-italic "P" icon.

## Quick start (contributor)

You need: macOS or Linux, Python 3.11+, Go, Bun (only for cm-bundle
rebuild), Temporal CLI, and a working Claude CLI logged into your
subscription.

```bash
git clone <repo>
cd prep-app
brew bundle           # macOS — installs python, go, temporal, bun, goreman
make setup            # venv + pip install + go build
make dev              # starts temporal + uvicorn + worker via Procfile
```

Open http://127.0.0.1:8081/ — you'll be auto-logged-in as
`dev@example.com` (the dev shim sets `PREP_DEFAULT_USER` so you don't
need Tailscale to develop).

`Ctrl-C` cleans up all three processes. See
[CONTRIBUTING.md](CONTRIBUTING.md) for everything else.

## Quick start (self-host on your tailnet)

This is the production-shape deploy: pm2 supervises uvicorn + worker +
temporal, Caddy reverse-proxies onto a path prefix, Tailscale Serve
fronts the box on HTTPS.

```bash
git clone <repo>
cd prep-app
make setup
# 1. install + start tailscale, run `tailscale up` once
# 2. install caddy, drop in the example config (see Caddyfile.example, Phase 1)
# 3. install pm2, run `pm2 start ecosystem.config.js` from a config you write
# 4. tailscale serve --bg --https=443 http://127.0.0.1:8000
```

Phase 1 of the OSS plan adds a docker-compose path that wraps these
steps. Until then, the CLAUDE.md / docs/oss-readiness.md notes have the
gory details.

## How auth works

Tailscale Serve sets a `Tailscale-User-Login` header on every request
that comes through `:443`. That's the user identity. Caddy's LAN
listener strips this header so a non-tailnet client can't spoof it.

There's no password / OAuth / magic-link path. Adding broader auth
isn't on the roadmap — the design is "people on your tailnet are the
user set."

For local development, `make dev` sets `PREP_DEFAULT_USER` to bypass
this. Don't ship that in prod.

## How agent calls work

The app shells out to your **local Claude CLI** for generation,
grading, and transform work — *not* to the Anthropic API directly.
Reasoning:

- No API-key management. Your existing CC subscription stays in your
  CLI's keychain.
- Local-first. Nothing crosses the network unless your CLI does it.
- Swappable. Other agent harnesses (`opencode`, `aider`, …) plug in via
  `PREP_AGENT_BIN` + `PREP_AGENT_ARGS` env vars.

You need a working `claude` CLI on PATH for the generation / grading /
transform paths to work. `make dev` won't fail without it; the routes
that require it will.
